Why This Workflow Matters

Here's the situation I see with most agencies and ghostwriters managing multiple LinkedIn clients:

You hop on a strategy call. The client drops 3-4 solid ideas, maybe shares a win, mentions something they're frustrated about. Good raw material. Then the call ends and you're staring at a 45-minute recording thinking "okay, now I need to turn this into 6 weeks of content."

So you manually transcribe (or wait for Fireflies to process), skim through looking for the gold, cross-reference against what you've already posted so you don't repeat topics, match everything to their brand voice, and format it into briefs your team can actually execute on.

That's 3-5 hours per client. Per call. And if you're running 8-10 clients, that's basically a full-time job just doing content brief admin.

The workflow I'm about to show you does the heavy lifting automatically:

1. Pulls the transcript from your call recording tool
2. Extracts the actual content ideas (not the small talk, not the scheduling discussion)
3. Checks against previous content to avoid repetition
4. Formats briefs that match each client's voice and content style
5. Drops everything into your project management tool, ready for review

You still review everything before it goes out. This isn't about removing human judgment - it's about removing the 3 hours of manual extraction and formatting that happens before you can even apply that judgment.

The Complete Workflow

Workflow Overview

``
[Fireflies Webhook] → [Extract Transcript] → [Filter for Content Ideas]
→ [Check Topic Database] → [Generate Briefs] → [Voice Check Agent]
→ [Create Tasks in ClickUp]
`

Total nodes: 12-15 depending on your setup

Section 1: Trigger and Transcript Extraction

Node 1: Webhook Trigger (Fireflies)

When Fireflies finishes processing a meeting, it fires a webhook with the transcript data. This is your trigger.

Configuration:
  • Webhook URL: Your n8n webhook URL
  • Authentication: Header auth (Fireflies sends a verification token)
  • Method: POST
What You Get: `json { "meeting_id": "abc123", "title": "Kyle / Client Name - Weekly Strategy", "transcript": "...", "speakers": ["Kyle", "Client"], "duration": 2847, "date": "2024-01-15" } `

Node 2: Set Client Context

Before you process anything, you need to know which client this is for. I use a simple lookup based on meeting title or participant names.

Configuration:
  • Function node that parses the meeting title
  • Matches against your client database (can be a simple JSON object or Airtable lookup)
  • Returns: client_id, client_name, voice_profile, content_themes, previous_topics
Tip: Keep your client profiles in a Google Sheet or Airtable base. Makes it easy to update without touching the workflow.

Section 2: Content Idea Extraction

Node 3: AI Content Extractor

This is where the magic happens. You're feeding the transcript to an AI model with very specific instructions about what to pull out.

Model: GPT-4 or Claude (I've been using Gemini lately for longer transcripts - better context window)

System Prompt:
`
You are a content strategist analyzing a client call transcript. Your job is to extract potential LinkedIn content ideas.

For each idea you identify, provide:
1. Topic summary (1 sentence)
2. Content angle (what makes this interesting/useful)
3. Supporting quotes from the transcript (exact quotes)
4. Content format recommendation (story post, how-to, opinion, case study)
5. Estimated engagement level (high/medium/low based on controversy, relatability, usefulness)

EXTRACT ONLY:
  • Client wins or case studies they mentioned
  • Problems they're solving for their customers
  • Opinions they expressed strongly
  • Frameworks or processes they described
  • Mistakes they admitted to or learned from
  • Industry observations or predictions
IGNORE:
  • Scheduling discussion
  • Small talk
  • Technical troubleshooting unrelated to content
  • Administrative items
Return as JSON array. `

`
Client: {{client_name}}
Industry: {{client_industry}}
Content themes they focus on: {{content_themes}}

{{transcript}}
`

Section 3: Topic Deduplication

Node 4: Check Against Previous Content

This is the part most people skip, and it's why their clients end up posting the same themes over and over.

You need a database of previous topics. I use Airtable for this, but you can use Google Sheets or even a simple JSON file stored in Google Drive.

Database Structure:
    • client_id: client_001 | topic_theme: automation ROI | post_date: 2024-01-08 | post_url: linkedin.com/... | performance: high
    • client_id: client_001 | topic_theme: process before tools | post_date: 2024-01-02 | post_url: linkedin.com/... | performance: medium
Node Configuration:
  • Pull last 90 days of topics for this client
  • Pass to AI for semantic similarity check (not just keyword matching)
Similarity Check Prompt: ` Compare these new content ideas against the client's recent topics.

New Ideas:
{{extracted_ideas}}

Recent Topics (last 90 days):
{{recent_topics}}

For each new idea, return:
  • is_duplicate: true/false
  • similar_to: [list of similar previous topics if any]
  • differentiation_suggestion: [how to angle it differently if it's similar but still usable]
We want to avoid repetition, but a topic can be revisited if the angle is genuinely different. `

Node 5: Filter and Flag

Function node that:
  • Removes duplicates marked as is_duplicate: true
  • Flags items that are similar but potentially usable with a different angle
  • Passes through completely fresh topics

Section 4: Brief Generation

Node 6: Generate Content Briefs

Now you're turning extracted ideas into actual briefs your team can execute on.

System Prompt:
`
You are creating a content brief for a LinkedIn ghostwriter. The brief should give them everything they need to write a post without additional research.

Client Voice Profile:
{{voice_profile}}

Brief Format:
1. POST CONCEPT: One sentence describing the post
2. HOOK OPTIONS: 3 different opening lines to test
3. KEY POINTS: 3-5 bullet points the post should cover
4. PROOF POINTS: Specific quotes, numbers, or examples to include
5. CTA SUGGESTION: What action should readers take
6. TONE NOTES: Any specific voice considerations for this topic
7. AVOID: Things that would sound off-brand or that we've overused

Keep briefs actionable. A writer should be able to draft from this in 30 minutes or less.
`

User Prompt:
`
Create a content brief for:
{{idea.topic_summary}}

Angle: {{idea.content_angle}}
Format: {{idea.format_recommendation}}
Source quotes: {{idea.supporting_quotes}}

Client typically writes about: {{content_themes}}
`

Output Example:
`
POST CONCEPT: Behind-the-scenes look at automating a proposal process that saved 40 hours/month

HOOK OPTIONS:
1. "We had three people touching every proposal. Now we have zero."
2. "40 hours a month. That's what was hiding in our proposal process."
3. "I automated myself out of a job. Specifically, the worst part of my job."

KEY POINTS:
  • Previous state: 3 people, manual handoffs, version control chaos
  • The trigger: realized they were spending more time on admin than actual proposal strategy
  • What they built: automated intake → template population → review queue
  • Results: 40 hours/month recovered, faster turnaround, fewer errors
  • What they do with the saved time now
PROOF POINTS:
  • "40 hours a month" (direct quote)
  • "three people touching every proposal" (direct quote)
  • "basically automated except for final review"
CTA SUGGESTION: Ask readers what process they'd automate first if time wasn't a factor

TONE NOTES: This client likes concrete numbers and behind-the-scenes honesty. Don't oversell the automation angle - focus on the practical outcome.

AVOID: Generic "automation is the future" framing. They've posted about this before. Keep it specific to this use case.
`

Section 5: Voice Quality Check

Node 7: Voice Check Agent

This is the quality gate. Before anything goes to your team, an AI agent reviews the brief to make sure it actually sounds like the client.

System Prompt:
`
You are a brand voice auditor. Your job is to review content briefs and flag anything that doesn't match the client's voice profile.

Client Voice Profile:
{{voice_profile}}

Review the brief for:
1. TONE MATCH: Does this sound like how the client actually talks? (yes/no/needs adjustment)
2. SPECIFICITY: Are there enough concrete details, or is it generic? (specific/needs more detail)
3. BRAND ALIGNMENT: Does this topic fit what they want to be known for? (aligned/off-brand/borderline)
4. AI TELLS: Does any language sound AI-generated rather than human? (clean/has tells)

If anything needs adjustment, provide specific revision suggestions.

Return: APPROVED or NEEDS REVISION with notes.
`

Branching Logic:
  • If APPROVED → proceed to task creation
  • If NEEDS REVISION → route to a review queue with the agent's notes
Important: Don't skip this step. TBH this is where most content automation falls apart - the brief technically works but sounds like ChatGPT wrote it. The voice check catches that before it wastes your writers' time.

Section 6: Output to Project Management

Node 8: Create ClickUp Task

Final step - create a task in ClickUp (or Notion, or Airtable, or wherever you manage content production).

Task Structure:
  • Title: [CLIENT NAME] - {{post_concept}}
  • Description: Full brief content
  • Custom Fields:
- Source: Strategy Call - {{meeting_date}} - Content Type: {{format_recommendation}} - Engagement Estimate: {{engagement_estimate}} - Voice Check Status: {{approval_status}}
  • Due Date: {{calculated based on your content calendar}}
  • Assignee: Default writer for this client
ClickUp API Configuration:
  • List ID: Your content production list
  • Space ID: Your client workspace
  • Authentication: API token

Configuration Notes

API Costs

Real talk on what this costs to run:

  • Fireflies: $10-19/month depending on plan
  • OpenAI API: ~$0.50-1.50 per call processed (varies by transcript length)
  • n8n: Free self-hosted or $20+/month cloud
  • Airtable: Free tier works for most agencies, Pro at $20/month if you need more
For an agency running 10 clients with bi-weekly calls, you're looking at maybe $40-60/month in API costs. Compare that to 40-60 hours of manual work.

Model Selection

I've tested this with:

  • GPT-4: Most reliable for brief generation, best at following complex instructions
  • GPT-4-Turbo: Faster, slightly cheaper, 90% as good
  • Claude 3 Opus: Excellent for voice matching, sometimes over-explains
  • Gemini 1.5 Pro: Best for very long transcripts (2hr+ calls), good value
For the content extraction step, Gemini's longer context window actually helps because you're not losing detail from longer calls.

For the voice check step, Claude tends to be more nuanced about catching "AI tells" in the writing.

Error Handling

Add these safety nets:

Node: Check Transcript Length
  • If transcript < 500 words, skip processing (probably a failed recording or wrong meeting)
Node: Fallback for API Errors
  • If AI API returns error, retry once, then create a manual review task instead of failing silently
Node: Rate Limiting
  • Add a 2-second wait between AI calls to avoid hitting rate limits during bulk processing

Customization Points

Different Transcription Services

Not using Fireflies? Here's how to adapt:

  • Otter.ai: Similar webhook structure, just different payload format
  • Fathom: Has direct Zapier integration, use Zapier to forward to n8n webhook
  • Rev.ai: API-based, you'd trigger manually or on schedule instead of webhook
  • Assembly AI: Great API, you can build the transcription into the workflow

Different Output Destinations

Notion:
  • Replace ClickUp node with Notion API
  • Create database item with brief as page content
  • Map custom fields to Notion properties
Airtable:
  • Direct Airtable node in n8n
  • Create record with brief in a long text field
  • Link to client record
Google Docs:
  • Create doc from template
  • Replace placeholders with brief content
  • Move to client folder

Adding More Context Sources

The workflow I showed you pulls from call transcripts. But you can feed the same extraction logic from:

  • Newsletter content the client sends you
  • Podcast appearances (get transcript, run through same flow)
  • Internal Slack channels where they share wins
  • CRM notes from sales calls
Same extraction prompt works - you're just changing the input source.

Troubleshooting

Problem: Briefs are too generic
Fix: Your voice profile isn't detailed enough. Add specific examples of how the client phrases things, words they use a lot, topics they always connect back to.

Problem: Getting too many duplicates flagged
Fix: Tighten your similarity threshold or add more nuance to the differentiation prompt. Some topics are worth revisiting - the check should catch exact repeats, not thematic overlap.

Problem: API costs higher than expected
Fix: Add a pre-filter that only processes calls tagged as "strategy" or "content planning." You don't need to run every internal sync through this.

Problem: Briefs don't match client voice
Fix: Your voice check agent needs better examples. Feed it 5-10 actual posts from the client so it knows what to compare against.
