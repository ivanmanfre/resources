# resources.ivanmanfredi.com

Source repo for Ivan Manfredi's lead-magnet library — interactive assessments, action checklists, automation ROI calculators, and playbooks for service-business operators at 11–100 employees.

## What's here

- **`index.html`** — browsable library index (auto-regenerated nightly from Supabase)
- **`sitemap.xml`** + **`robots.txt`** — SEO plumbing (auto-regenerated nightly)
- **`.nojekyll`** — disables Jekyll processing so `_engine/*` paths serve verbatim
- **`_engine/`** — shared renderers for the 3 interactive formats:
  - `checklist.{js,css}` — interactive checklist with localStorage + email-gated personalized follow-up
  - `calculator.{js,css}` — live-compute calculator with tier pills + conditional recommendations
  - `assessment.{js,css}` — multi-step assessment with per-category scoring, partial-reveal email gate, LinkedIn share
  - `demo/{checklist,calculator,assessment}/` — live engine demos with realistic sample data
- **`assets/og/`** — 1200×630 OpenGraph PNGs for every LM (branded with Ivan's photo + format badge + title)
- **`<slug>/index.html` + `<slug>/data.json`** — per-LM wrapper + content spec (new engine-based LMs)
- **`<slug>/index.html`** (standalone) — legacy full-HTML LMs (guides, swipe files)

## Architecture

All content here is auto-generated by n8n workflows backed by Claude (via a Railway proxy). The library follows a three-engine architecture:

1. **Checklist** → Claude emits a JSON spec → thin wrapper loads `/_engine/checklist.js` + `./data.json`
2. **Calculator** → Claude emits a JSON spec → thin wrapper loads `/_engine/calculator.js` + `./data.json`
3. **Assessment** → Claude emits a JSON spec → thin wrapper loads `/_engine/assessment.js` + `./data.json`

Guides and swipe files are generated as monolithic HTML (no engine).

Every page fires a beacon to a Supabase edge function on view, capture, completion, CTA click, and share. Data lands in `lm_events` and `assessment_results` tables for conversion tracking.

## Contributing

This is a content repo — not for external PRs. Everything is regenerated by the Lead Magnets n8n workflow on ClickUp status transitions.

To re-render any existing LM with the latest engine + brand:

```bash
curl -X POST https://n8n.ivanmanfredi.com/webhook/regen-lm \
  -H "Content-Type: application/json" \
  -d '{"slug":"<slug>"}'
```

The re-render bypasses the post/email side-effects of a full ClickUp status flip — it only regenerates the HTML + data.json.

To re-push the library index, sitemap, engine files, and OG mirrors:

```bash
curl -X POST https://n8n.ivanmanfredi.com/webhook/publish-library
```
