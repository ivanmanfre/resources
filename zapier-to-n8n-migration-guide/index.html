Why This Migration Matters

Zapier is excellent for validation. Point, click, connect, done. But once you run a real operation — 30+ zaps, multi-step branching, data transforms, error handling — the model works against you. Per-task pricing means your automation bill scales with your success, not your infrastructure.

I audited one property management client last quarter: 47 active zaps, 92,000 tasks/month, $679/month Zapier bill. We moved the whole estate to n8n on a $40/month VPS. Same workflows, more reliable error handling, and the team actually understood what was happening under the hood because they could read the nodes.

If you're paying more than $200/month on Zapier and you have an ops person who can read JSON, you're leaving money and capability on the table. The migration pays for itself inside 90 days.

1. Audit Your Current Zapier Setup

You cannot migrate what you haven't mapped. Before you touch n8n, spend two hours cataloging everything currently running in Zapier. This is the step most teams skip, and it's why migrations stall at 40% complete.

Step 1: Build the Zap Inventory

Go to your Zapier dashboard, filter to active zaps, and document every single one. A spreadsheet with these columns is enough:

Zap Name Trigger App Action Apps Monthly Tasks Criticality Complexity
New Lead to Slack HubSpot Slack, Google Sheets ~200 High Low
Invoice Created Xero Gmail, Notion ~50 High Medium
Form to CRM Typeform HubSpot, Slack ~300 Medium Low
Client Onboarding Sequence Stripe HubSpot, ClickUp, Gmail, Slack ~40 Critical High

Step 2: Identify Your Core Integrations

Look at the inventory and count app occurrences. The top 5-7 apps are your core integrations — these must work flawlessly in n8n before anything else moves. Patterns I see across service businesses:

  • CRM: HubSpot, Salesforce, Pipedrive, Close.
  • Accounting: Xero, QuickBooks, Stripe, Chargebee.
  • Communication: Slack, Gmail, Microsoft Teams, Outlook.
  • Project management: ClickUp, Asana, Notion, Monday.
  • Forms & intake: Typeform, Tally, Google Forms, Jotform.
  • Telephony & support: Aircall, Intercom, Zendesk, Front.

Step 3: Flag the Complex Ones

Some zaps will port in 10 minutes. Others will take half a day because they lean on Zapier-specific features. Flag any zap that:

  • Uses Paths, Formatter, Filter, or Sub-Zap chains.
  • Has more than 5 steps or multiple branches.
  • Connects to a niche app (check n8n's integrations list first).
  • Handles invoicing, onboarding, or anything touching money.

Do not migrate randomly. I've seen a staffing firm break payroll notifications because they moved "the easy ones first" and missed a dependency on a Zapier Formatter step upstream.

2. Score & Prioritize the Migration Order

Not every zap deserves to survive. Roughly 20-30% of what I find in client Zapier accounts is abandoned, redundant, or better solved by a native product feature. Use this rubric to score each zap before you migrate it.

Score Keep in Zapier (temporarily) Migrate to n8n Delete or replace
Task volume < 50/mo 50-10,000/mo 0 in last 60 days
Business value Low but still used High, mission-critical Legacy, unowned
Complexity Single-step, obscure app Multi-step, core apps Duplicated elsewhere
Cost driver Negligible Top 10 by task usage N/A

Rank the "migrate" column by cost impact first. The top 5 zaps by task volume usually account for 60-75% of your Zapier bill. Move those first and you'll see most of the savings inside a month.

3. Stand Up Your n8n Infrastructure

You have three deployment options. Pick based on your team's comfort with servers, not based on what sounds cool.

Option Monthly cost Setup time Best for Trade-off
n8n Cloud (hosted) $24-120 10 min No DevOps resource, < 10k executions/mo Still limited by execution tier
Self-hosted VPS (Hetzner, DigitalOcean) $20-60 2-3 hrs Most 11-100 employee businesses You own uptime and backups
Self-hosted on your cloud (AWS, GCP) $40-200 4-8 hrs Regulated industries, internal IT team Higher ops overhead

For 90% of service businesses I work with, a Hetzner CX22 ($8/month) plus managed Postgres on Supabase ($25/month) handles 100,000+ executions comfortably. Add a domain, Cloudflare in front, and you're done.

The minimum viable setup checklist

  • Docker Compose deployment — the official n8n-docker-caddy template works out of the box.
  • External Postgres — never use SQLite in production, you will lose data on restart.
  • Automated daily backups — pg_dump to S3 or Backblaze B2 ($0.005/GB).
  • TLS via Caddy or Traefik — not optional, your webhooks are exposed.
  • Basic auth or SSO — lock down the editor behind Cloudflare Access if you can.
  • Execution pruning — set EXECUTIONS_DATA_PRUNE=true, or your DB will balloon.

4. Map Zapier Concepts to n8n Equivalents

Most migration mistakes come from trying to rebuild a zap one-for-one. n8n thinks about workflows differently, and once you see the mapping, the rebuild goes fast.

Zapier concept n8n equivalent What changes
Trigger Trigger node (Webhook, Schedule, App Trigger) Webhooks are first-class and free
Action step Any integration node or HTTP Request Any REST API works, no "premium app" gating
Filter IF node or Filter node Can branch true/false, not just block
Paths Switch node Unlimited paths, not capped at 3
Formatter Set node, Code node, or built-in expressions JavaScript is available everywhere
Delay Wait node Wait for webhook, time, or external event
Digest / Storage Postgres, Supabase, or n8n static data You own the storage layer
Sub-Zap Execute Workflow node Cleaner, supports item-by-item execution

The expressions shift

Zapier uses {{1.field_name}}. n8n uses {{ $json.field_name }} or {{ $node["NodeName"].json.field }}. Once you're past that, the mental model is nearly identical. The bonus: n8n expressions are full JavaScript, so {{ $json.email.toLowerCase().trim() }} just works without a separate Formatter step.

5. The Actual Migration Playbook

Here's the sequence I follow with every client. Do not reorder the steps — the order is what protects you from breaking production.

  1. Rebuild one non-critical zap first. Pick something low-volume and low-stakes. Get the connection, auth, and trigger working. This is your learning curve, do not burn it on the invoice workflow.
  2. Run both systems in parallel for 7-14 days. Turn the n8n workflow on alongside the Zapier one, but route its output to a test Slack channel or a staging record. Compare outputs daily.
  3. Switch the trigger, not the target. When you cut over, change where the trigger fires. Don't try to disable the Zap and enable n8n at the same moment — you'll lose events in the gap.
  4. Leave the Zapier zap disabled (not deleted) for 30 days. If anything breaks, you can re-enable in 30 seconds while you diagnose.
  5. Migrate in waves of 3-5 workflows per week. Not all at once. Your ops team needs time to report issues, and you need time to fix them before the next wave.
  6. Decommission Zapier only after 60 days of clean parallel runs. Then downgrade the plan or cancel. Most teams discover one or two zaps they forgot about in week 3 — the grace period saves you.

The single biggest migration failure mode is ego: "we'll just cut over this weekend." Every team that tries this loses something. Stagger the work, keep Zapier alive as a safety net, and you'll sleep through the cutover.

Where self-hosting wins beyond cost

  • No task limits: run a 500,000-execution backfill without a bill shock.
  • Real error handling: retries with backoff, error workflows, custom alerting to Slack or PagerDuty.
  • Version control: workflows are JSON, commit them to Git, review changes in PRs.
  • AI-native nodes: LangChain, Claude, OpenAI, and vector DB nodes ship in the core product.
  • Custom code anywhere: drop into a Code node and call any library when the UI isn't enough.

7. The Traps Nobody Warns You About

I've made every one of these mistakes so you don't have to.

  • Credential sprawl: you'll re-auth every integration. Budget 2-3 hours just for OAuth flows with HubSpot, Google, and Slack. Do this on a Friday afternoon, not cutover morning.
  • Webhook URL changes: Typeform, Stripe, Calendly and similar services need their webhook URLs updated when you move. Make a list before you migrate, update in one sitting.
  • Rate limits you never noticed: Zapier silently throttles. n8n will happily fire 200 HubSpot calls in a second and get 429'd. Add a Split In Batches node with 10-item batches and 1-second waits.
  • Item-by-item vs. bulk execution: n8n processes each item through the whole workflow by default. A Zap that "runs once per new row" may need a Loop or SplitInBatches pattern in n8n to match behavior.
  • Empty results break workflows: Zapier stops silently. n8n throws. Add an IF node checking {{ $json.length > 0 }} before downstream steps.
  • No built-in storage: Zapier's Storage by Zapier has no direct n8n analog. Use Supabase, Postgres, or n8n's static workflow data for state between executions.
  • Error workflows are opt-in: set a dedicated error workflow at the instance level so every failed execution pings Slack. Without it, you find out about broken automations from an angry client.

One last thing: document your n8n workflows with sticky notes inside the canvas. Future-you, or whoever inherits this six months from now, will need to understand the branching logic at a glance. n8n lets you annotate — use it.
