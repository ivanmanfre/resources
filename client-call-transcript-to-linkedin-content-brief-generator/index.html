Why This Workflow Matters

Here's what I see constantly with marketing agencies and content strategists: you're on calls with clients all day, capturing genuinely useful insights, and then... those insights sit in transcripts nobody reads.

The manual process looks like this:
1. Call ends
2. Transcript lands in Fireflies
3. Someone (usually you) has to carve out 45-60 minutes to review it
4. Pull out the relevant talking points
5. Figure out which ones are actually worth turning into content
6. Write up the brief
7. Send it somewhere for approval
8. Repeat for every single client call

That's 3-5 hours per week minimum just on the extraction and brief writing part. And that's being generous.

The workflow I'm giving you automates steps 2-6 entirely. You go from "call ended" to "content brief in ClickUp" without touching anything in between.

Workflow Architecture Overview

Trigger: New transcript appears in Fireflies (or your transcription tool)

Stage 1: Content extraction - AI pulls potential content angles from the transcript

Stage 2: ICP relevance filter - Agent checks if the content actually matters to your audience

Stage 3: Tone and voice application - Rewrites to match your brand voice

Stage 4: Quality check - Makes sure it doesn't sound like AI wrote it

Stage 5: Delivery to ClickUp with scheduling options

Let me walk you through each piece.

Stage 1: Transcript Ingestion and Content Extraction

Trigger Configuration

Node Type: Webhook or Fireflies Integration

What It Does: Listens for new transcripts from your meeting recording tool

Setup Notes:
  • If using Fireflies, you can trigger on "meeting completed"
  • Pull the full transcript text plus meeting metadata (date, attendees, meeting title)
  • Store the client name from the meeting title or attendee list - you'll need this for tone matching later

Content Extraction Agent

Node Type: AI Agent (GPT-4 or Claude)

System Prompt:

``
You are a content strategist reviewing client call transcripts. Your job is to identify 3-5 potential LinkedIn content angles from this conversation.

For each angle, extract:
1. The core insight or talking point
2. A potential hook (first line that would grab attention)
3. The target emotion or reaction you want from readers
4. Any specific data points, metrics, or examples mentioned
5. Content format recommendation (story post, tactical breakdown, contrarian take, case study snippet)

Only extract insights that would be relevant to [ICP DESCRIPTION]. Ignore small talk, scheduling discussions, and administrative content.

If the transcript contains fewer than 2 viable content angles, flag it as "low content value" and explain why.
`

Input: Full transcript text

Output: Structured JSON with content angles

Stage 2: ICP Relevance Filter

This is the step most people skip, and it's why their content machines pump out garbage.

Relevance Analysis Agent

Node Type: AI Agent

System Prompt:

`
You are a content quality gatekeeper. Review each content angle and determine if it passes the ICP relevance test.

ICP Profile:
[INSERT YOUR ICP DESCRIPTION - job titles, industries, pain points, what they care about]

For each content angle, answer:
1. Would this resonate with the ICP? (Yes/No/Maybe)
2. Confidence level (High/Medium/Low)
3. If No or Maybe, explain what's missing or off-target
4. Suggested modifications to make it more relevant (if applicable)

Only pass through angles with "Yes" and "High" or "Medium" confidence.
`

Decision Logic:
  • Yes + High Confidence → Proceed to Stage 3
  • Yes + Medium Confidence → Proceed with modifications
  • Maybe or Low Confidence → Hold for manual review
  • No → Discard with reason logged
This filtering step is what separates a content machine that produces usable output from one that just creates more work.

Stage 3: Tone and Voice Application

Voice Transformation Agent

Node Type: AI Agent

System Prompt:

`
You are a ghostwriter applying a specific brand voice to content briefs.

Voice Profile:
[INSERT YOUR VOICE DOCUMENT OR KEY CHARACTERISTICS]

Take the content angle provided and write:
1. A full LinkedIn post draft (150-300 words)
2. 3 alternative hooks
3. A suggested CTA

Maintain the core insight but transform the language to match the voice profile exactly. The output should sound like [CLIENT NAME] wrote it, not like AI generated it.

Avoid: [LIST YOUR FORBIDDEN WORDS/PHRASES]
`

Input: Approved content angles from Stage 2

Output: Draft LinkedIn posts with alternatives

Stage 4: AI Detection Quality Check

This is the agent that makes sure your content doesn't scream "I used ChatGPT for this."

Quality Assurance Agent

Node Type: AI Agent

System Prompt:

`
You are an AI content detector working in reverse. Your job is to identify and fix anything that makes this content sound AI-generated.

Check for:
1. Overused transitional phrases ("Here's the thing", "Let me break this down")
2. Generic business buzzwords ("leverage", "synergy", "game-changer")
3. Formulaic structure (hook → lesson → CTA with no variation)
4. Lack of specific details, numbers, or examples
5. Sentences that sound like a textbook instead of a person

For each issue found:
  • Flag the specific text
  • Explain why it's a problem
  • Provide a rewritten alternative
Output the final cleaned version plus a quality score (1-10).

Only pass content with a score of 7 or higher. Anything below needs manual review.
`

Threshold Logic:
  • Score 8-10 → Auto-approve, send to ClickUp
  • Score 7 → Send to ClickUp with "light review needed" tag
  • Score below 7 → Send to manual review queue

Stage 5: ClickUp Delivery and Scheduling

Task Creation Node

Node Type: ClickUp Integration

Task Fields:
  • Title: [Client Name] - LinkedIn Post: [Hook Preview]
  • Description: Full post draft + alternatives + source transcript link
  • Status: "Ready for Review" or "Approved" based on quality score
  • Due Date: Suggested posting date based on content calendar logic
  • Custom Fields:
- Content Source: Call transcript - Quality Score: [from Stage 4] - Content Type: [from Stage 1 recommendation] - Client: [extracted from transcript]

Optional: Scheduling Extension

If you want to take it further, add a node that:
1. Checks your content calendar for open slots
2. Suggests optimal posting times based on past engagement data
3. Auto-schedules approved content (score 8+) to your posting tool

Extension: Multi-Source Content Machine

Once the basic workflow is running, you can expand the trigger sources. Instead of just Fireflies, pull from:

  • LinkedIn profiles of competitors or thought leaders - Scrape their top-performing posts (filter by engagement metrics), run through the same pipeline
  • Industry newsletters - Forward them to a dedicated email, trigger extraction on new messages
  • Your own content performance data - Feed back what's working into the system to improve format recommendations
The key insight from building this for clients: the scraping should only grab content that's already performing well. Set a threshold for impressions or comments so you're learning from what actually resonates, not just what gets posted.

Short-Form Content Extension

Here's the bonus that clients love: when a LinkedIn post performs well (hits your engagement threshold), trigger a secondary workflow that:

1. Takes the successful post
2. Generates a short-form video concept based on it
3. Outputs a script, shot list, and hook variations
4. Delivers to your video production queue

You're basically turning one call insight into multiple content formats without any additional manual work after the initial approval.

Common Pitfalls to Avoid

Skipping the ICP filter. I see this constantly. People build the extraction and generation parts, then wonder why 70% of their output is irrelevant. The filter isn't optional - it's what makes the system actually useful.

Not having a real voice document. "Professional but friendly" isn't a voice profile. You need specific phrases you use, words you avoid, sentence patterns that sound like you. The more detailed your voice input, the better your output.

Auto-publishing without review. Even with a quality check agent, keep a human in the loop for final approval. The goal is to reduce work from 3-5 hours to 15-20 minutes of review, not to eliminate human judgment entirely.

Ignoring the feedback loop. Track which auto-generated posts perform well and which flop. Feed that data back into your prompts. The system should get better over time, not stay static.

Credential Setup Guide

This workflow requires the following credentials. Here is how to set up each one:

1. Anthropic (Claude AI)

Go to console.anthropic.com/settings/keys and create an API key. In n8n, add an Anthropic credential and paste the key.

2. Google (Drive/Docs/Sheets)

Use OAuth2 authentication. In n8n, add a Google OAuth2 credential. Create a project in Google Cloud Console, enable the relevant APIs, and configure OAuth consent screen + credentials.

3. Slack

Create a Slack App at api.slack.com/apps. Add required scopes (chat:write, channels:read). Install to workspace and copy the Bot Token into n8n.

4. Webhook

No external credentials needed. n8n generates the webhook URL automatically. Ensure your n8n instance is publicly accessible so external services can reach it.

5. HTTP Request

Depends on the API being called. Check the specific API documentation for authentication method (API key, OAuth2, Bearer token) and configure the appropriate credential type in n8n.
