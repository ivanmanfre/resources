This checklist fixes that.

What To Do With This

Spend 30-45 minutes going through this checklist BEFORE you scope any document automation project. Bring in whoever owns IT/infrastructure for 15 minutes of that time. Fill out each layer honestly.

If you hit more than 3 red flags, you need to scope the access layer as a separate phase with its own timeline and budget. Seriously. This is the difference between a 2-week project and an 8-week project.

Why This Matters

Most document automation projects fail before the automation even starts. Not because the automation is hard - that part is usually straightforward. They fail because nobody scoped the access layer properly.

I've worked with insurance companies where the documents live on cloud servers behind VPNs. Construction firms where PDFs are 300 megabytes and crash everything. Professional services teams where three different people need to sign every page of a document.

The pattern is always the same: the client says "we need to automate document signing" and everyone starts talking about DocuSign vs Adobe Acrobat. Meanwhile, nobody's asked how the automation will actually reach the files in the first place.

Access issues typically add 2-4 weeks to project timelines. But here's the thing - if you identify them upfront, you can plan for them. If you discover them at week two, you're scrambling.

The 3-Layer Checklist

Layer 1: Document Access

Before anything else, you need to know how files will flow into the automation.

Answer these questions:

  • [ ] Where do the documents currently live? (local server, cloud storage, email attachments, software export)
  • [ ] What's required to access them? (VPN, username/password, specific network location, office-only access)
  • [ ] Can documents be accessed from outside the office? If yes, what's required?
  • [ ] Who currently has access to these documents? Is there a permissions layer?
  • [ ] What format are the documents in? (PDF, Word, Crystal Reports export, etc.)
  • [ ] What's the typical file size? (Under 10MB, 10-50MB, 50-100MB, 100MB+)
  • [ ] Are there documents that regularly crash or slow down computers when opened?
Red flags that add 2+ weeks:
  • VPN required with no API or remote machine option
  • Documents live on local servers with office-only access
  • Files regularly exceed 100MB
  • Access requires manual steps that can't be automated (like physical tokens)
Your access situation:
    • Access Type: Cloud storage with API | Timeline Impact: Minimal | Solution Options: Direct integration
    • Access Type: Cloud storage, VPN required | Timeline Impact: +1-2 weeks | Solution Options: Remote machine, RPA, or VPN automation
    • Access Type: Local server, office-only | Timeline Impact: +2-4 weeks | Solution Options: Remote desktop, server-side agent, or workflow redesign
    • Access Type: Mixed/unclear | Timeline Impact: +2-4 weeks | Solution Options: Discovery phase required before scoping

Layer 2: Document Processing

Once you can access the files, what needs to happen to them?

Answer these questions:

  • [ ] What triggers the document workflow? (email, software event, manual upload, scheduled time)
  • [ ] Does the document need to be read/parsed by AI? If yes, what data needs to be extracted?
  • [ ] Does the document need signatures? How many signers?
  • [ ] Do all signers sign the same pages, or different pages?
  • [ ] Does every page need a signature, or just specific pages?
  • [ ] What happens after processing? (store back in folder, email to someone, update a system)
  • [ ] Does the processed document overwrite the original or create a new version?
Red flags that add complexity:
  • Multiple signers who each need to sign every page
  • Large documents where AI needs to parse specific sections (not the whole thing)
  • Documents that need to be saved back to the original secure location
  • Processing triggers that are inconsistent or manual
Signature workflow specifics:
    • Signature Scenario: Single signer, end of document | Complexity: Low | Recommended Tool: Most e-sign tools work
    • Signature Scenario: Multiple signers, same pages | Complexity: Medium | Recommended Tool: DocuSign, Adobe Sign with templates
    • Signature Scenario: Multiple signers, every page | Complexity: High | Recommended Tool: Requires careful template setup, test thoroughly
    • Signature Scenario: Certification without signatures on all pages | Complexity: Problem | Recommended Tool: This is actually a compliance risk - need to fix process

Layer 3: Infrastructure & Integration

This is where projects usually stall. The automation is ready, but it can't talk to your systems.

Answer these questions:

  • [ ] Does your document software have an API? (e.g., Adobe Acrobat Sign API, DocuSign API)
  • [ ] Do you have access to create API credentials, or does IT need to provision them?
  • [ ] How long does IT typically take to provision new system access? (days, weeks, requires committee approval)
  • [ ] Are there remote machines/servers available that could host automation tools?
  • [ ] What automation platform will orchestrate the workflow? (n8n, Make, Zapier, custom)
  • [ ] Do you have existing automation infrastructure, or is this the first project?
  • [ ] Who will maintain the automation after it's built?
Red flags that add 2+ weeks:
  • No API available for key systems
  • IT provisioning takes more than 1 week
  • No remote machine option for secure access scenarios
  • First automation project with no existing infrastructure
  • Unclear ownership for ongoing maintenance
Integration reality check:
    • Your Situation: API available, credentials accessible | What This Means: You can integrate directly - cleanest option
    • Your Situation: API available, IT provisioning slow | What This Means: Factor in 1-2 week wait time, or start the request NOW
    • Your Situation: No API, but browser-based workflow | What This Means: RPA (browser automation) is possible but adds complexity
    • Your Situation: No API, no browser option | What This Means: You might need to redesign the process, not just automate it

The Pre-Project Diagnostic

Before you get a quote or timeline from anyone, score yourself:

Layer 1 (Document Access)
  • All documents accessible via API or cloud storage with no VPN: 0 points
  • VPN required but remote machine available: 1 point
  • VPN required, no remote machine, needs investigation: 2 points
  • Office-only access, unclear path forward: 3 points
Layer 2 (Document Processing)
  • Simple trigger, simple processing, straightforward output: 0 points
  • Multiple signers or AI parsing required: 1 point
  • Complex signature requirements or large file handling: 2 points
  • Multiple red flags or unclear requirements: 3 points
Layer 3 (Infrastructure & Integration)
  • APIs available, credentials accessible, existing automation platform: 0 points
  • APIs available but IT provisioning needed: 1 point
  • No API, will need RPA or workarounds: 2 points
  • First automation project, no infrastructure, unclear IT timeline: 3 points
Your score:
    • Total Score: 0-2 | What It Means: Standard project | Timeline Adjustment: Scope as quoted
    • Total Score: 3-5 | What It Means: Access layer needs attention | Timeline Adjustment: Add 1-2 weeks, separate discovery phase recommended
    • Total Score: 6-7 | What It Means: Significant infrastructure work | Timeline Adjustment: Add 2-4 weeks, definitely scope access layer separately
    • Total Score: 8-9 | What It Means: Major project, not just automation | Timeline Adjustment: Full discovery engagement before any automation scoping

Implementation Priority

If you scored 3 or higher, do these three things before you scope the automation:

1. Get IT in the room for the scoping conversation. Not after. During. The number of projects I've seen stall because "we didn't realize IT needed to approve that" is embarrassing.

2. Test the access path manually. Can you actually get to the documents the way the automation would need to? Do it yourself, document the steps, note every login, VPN connection, and permission required.

3. Separate the access layer into its own phase. Give it its own timeline and budget. When you lump it into "the automation project," it's the part that always bleeds over.

Your Next 7 Days

Day 1-2: Fill out this checklist completely. Don't guess - actually verify each answer.

Day 3: Schedule 30 minutes with whoever owns IT/infrastructure. Walk through Layer 3 together.

Day 4-5: Test the access path manually. Document exactly what it takes to get to one document and process it.

Day 6-7: Calculate your score. If it's 3+, draft a separate access layer scope before you talk to any automation consultant.

The automation part of document automation is usually the easy part. The hard part is everything that happens before the automation can even start. This checklist makes sure you know that before you're 3 weeks in and 40% over budget.
